## 0. Product framing (READ FIRST — authoritative; overrides any quant-"flagship" language below)

**What Calma is.** Independent verification by *execution to ground truth*. It does not read code and form
an opinion (that is an eval/judgment tool) — it **re-executes the work against the data and recomputes the
truth from raw outputs**, then proves or breaks the claim. Independence is the principle: the system that
produced a result can't be trusted to verify it (producer ≠ verifier).

**What THIS build is — the skill (domain-agnostic, agent-callable).** A general Claude Code skill that
verifies **AI-agent-produced results** by re-executing them to ground truth. **No target domain.** Used
two ways: **post-hoc** (verify what an agent just produced) and **inline** (an agent calls it as a
guardrail while it works). The quant/DS depth (Sharpe, DSR, backtest realism, fills/holdout-on-a-quarter)
belongs to the **CLI end-goal, NOT this skill.** Anywhere the body below calls quant the "flagship" or
bakes quant statistics into the core, read that as **"quant is ONE optional domain pack"** and treat the
deep quant stats as CLI-layer content.

**The method = the four checks, generalized** (lifted off any one domain):
1. **Provenance / no-leakage** — every input to the result was knowable at the time it was used (general
   form of point-in-time / look-ahead).
2. **Independent recomputation** — recompute the headline number from raw machine-readable outputs, never
   the reported value and never by reading the producing code.
3. **Unseen-data re-run** — re-execute the frozen artifact on data withheld from its construction
   (overfitting/generalization; applies wherever there is a fitted/predictive component).
4. **Invariant assertion** — the identities/properties that must hold for an honest result; surface the
   first violation.

Check availability: **universal** = reproducibility + independent recomputation (always). **Domain-
pluggable** = holdout (fitted/predictive work), point-in-time (time-series), plus declared invariants per
domain. The more general the target, the more it leans on the universal two + declared invariants; domain
packs add the sharper checks.

**Two invocation modes the skill must support:**
- **Post-hoc** — heavier, thorough: full re-execute + holdout where applicable.
- **Inline / continuous** — fast guardrail an agent calls mid-work; returns a crisp, machine-consumable
  verdict it can branch on. **Tiered:** cheap checks (invariants, static provenance, cached reproduce)
  inline; reserve full re-execution for post-hoc.

**Agent-callable contract.** The primary caller is another agent. The verdict is **exit-code-style +
structured** (PASS=CONFIRMED / CONFIRMED-WITH-CAVEATS / FAIL=REFUTED / INCONCLUSIVE + the one reason), and
verification works **with an explicit claim (claim-vs-recompute) OR without one (property / invariant /
reproducibility checks)** — not every check needs a stated number.

**Independence under self-invocation.** When the agent that produced the result also invokes Calma,
independence is preserved by the **deterministic substrate, not by who calls it**: the model only proposes
bindings/claims; scripts compute every statistic and the verdict. A producing agent cannot steer a
script-computed verdict.

**Honest limits (general/agent case).** Execution-to-ground-truth proves *the stated result is real and
reproduces* — not that the agent did the *right task* or that the approach is sound; state this in scope.
Much agent/LLM-driven output is nondeterministic and won't reproduce exactly → **INCONCLUSIVE (or property-
based checks) is the common, correct outcome there — never a false FAIL.**

## 0.5 Product decisions for adoption, value & quality (how the gaps are resolved)

**North star: Calma is the trust layer for agentic work.** As people delegate real work to coding/research
agents, they have no reliable way to know if an agent's results are real. Calma re-executes to ground truth
so you can **trust — or catch — what your agents produce.** Because the **agent is the producer and the
human is the user, a FAIL is pure value to the user, not an ego hit** (it caught the agent's mistake before
they shipped it). This reframes verification from a vitamin into a painkiller and dissolves the
"nobody-wants-to-be-told-they're-wrong" adoption problem that sinks self-audit tools.

Decisions (each = a gap resolved + the adoption/value/quality reason it's made this way):

1. **Zero-config, instant value [adoption].** Bare `verify <target>` produces a real verdict with no
   setup. `draft_contract.py` auto-detects entrypoint/outputs/metrics AND **auto-extracts the claim from
   the agent's own output/conversation** ("the agent reported AUC 0.94", "all tests pass", "processed 10k
   rows"). Friction is the #1 killer of a free skill — the default path is one command, one confirm.

2. **Always-on default checks that deliver value on ANY result [value; resolves gap 1].** No-config,
   always run: **(a) does it reproduce** (re-run → same number), **(b) independent recomputation**
   (recomputed metric vs the claim), **(c) leakage heuristics.** These catch the most common, most
   expensive agent lies (number doesn't reproduce; recomputed ≠ reported; obvious leakage) cheaply — so
   value lands even when full re-execution or holdout isn't possible. Domain packs + declared invariants
   add depth for power users. Honest scope: this proves the result is REAL and reproduces, not that the
   agent did the *right task*.

3. **Verify with OR without a stated number [value; resolves gap 2].** Claim present → recompute-and-diff.
   No claim → property / invariant / reproducibility checks. Claim auto-extraction keeps the common case
   zero-effort.

4. **Two modes, tiered for cost [adoption+value; resolves gap 3].** Post-hoc = thorough (full re-execute +
   holdout). Inline = fast guardrail an agent calls mid-work; **only re-verifies what changed (cached)**,
   so continuous use is cheap enough to leave on.

5. **Verdict computed by code, not opinion [trust = the differentiator; resolves gap 4].** Deterministic
   scripts compute every statistic and the verdict; the model only proposes bindings/claims. **Even the
   agent that wrote the code can't talk Calma out of a FAIL.** This is the one-line reason Calma beats
   "just ask the agent to check it."

6. **INCONCLUSIVE is actionable, never a shrug [quality+trust; resolves gap 5].** When it can't fully
   verify (nondeterminism, missing seed, un-emitted outputs), it returns **what IS verified** + the
   **exact fix** to make it verifiable ("set seed=...", "emit metrics.csv"). Bias to CAVEAT/INCONCLUSIVE
   over a false FAIL; every FAIL ships a reproduction. One false FAIL burns trust; a useful INCONCLUSIVE
   keeps it.

7. **Every FAIL is a shareable teardown [growth engine].** A catch outputs a reproducible "claimed X →
   really Y, here's why, here's the repro command" (the BTC-teardown shape). Users share catches → the
   value is self-evident → installs follow. The distribution loop is built into the output format.

8. **Scope honesty as a feature [quality+trust].** Every verdict stamps WHAT was verified and what wasn't.
   Honest scope is what makes a verifier credible; overclaiming is what kills one.

Build emphasis (no descoping — ordering for value): the **adoption core** — zero-config claim
auto-extraction, the three always-on default checks, actionable INCONCLUSIVE, and the shareable-FAIL output
— are **first-class M1 requirements**, because they are what win the first users. The heavier quality/trust
machinery (verified isolation tier, pinned libm, DSR/PBO) is still built in full; it deepens trust but is
not what lands day-one value.

## 1. Product thesis

Calma Stage-1 is a Claude Code skill that does for COMPUTATIONAL RESULTS what paper-audit does for prose: it adversarially RE-RUNS a piece of work inside a verified isolation boundary, RECOMPUTES the headline number from raw machine-readable outputs using CALMA'S OWN reference-deterministic recipe (never the author's reported value, never the author's recipe when a canonical one exists), over an INDEPENDENTLY-VALIDATED, semantically-typed input binding, DIFFS it against the human's pinned claim under a STATISTICALLY-CALIBRATED tolerance MODEL that pools numeric-reproducibility spread, cross-stack budget, author/claim precision AND the claimed statistic's own SAMPLING standard error, then hunts the ways empirical results lie across FIVE failure families (leakage, overfitting/selection, realism gap, data integrity, baseline/comparative-validity) and emits a fixed-vocabulary, falsifiable, per-claim + repo-level verdict. The product is TRUST, not confidence: the verdict ENUM is computed deterministically by a single shared verdict() pure function (never the model), every REFUTED ships a re-runnable reproduction whose machine-checkable `expected` assertion IS the refutation, tied to a content-addressed SBOM manifest, and every ambiguity (low-coverage band, unbound input, no isolation primitive, resource-kill, path-dependence, NaN/degenerate recompute, unconfirmed claim target) degrades to CAVEAT/INCONCLUSIVE — never a false REFUTED. Honesty invariants preserved and hardened, ALL machine-enforced not prose: no statistic OR verdict label computed by the model (the model only PROPOSES bindings/claim targets, which are deterministically type-checked AND user-confirmed before any REFUTED); install AND run AND every interpreter startup are untrusted-code execution behind the SAME verified isolation tier; no REFUTED under uncontrolled-and-insufficient-K determinism or resource-kill; no auto-inferred trial-count N; recompute only from machine-readable raw outputs in Calma's deterministic float64+pinned-libm reference path; canonical-recipe authority bounded by a co-equal-convention set (out-of-set alone can only CAVEAT until the set is review-complete); independent input-binding gating EVERY REFUTED axis; gate-and-verdict authority owned by scripts. Flagship: quant-backtest verification (effective-N Deflated Sharpe + realism deflators + trivial-baseline edge), a specialization of one general engine — at M1 it delivers the BREAK-IT half (REFUTED via baseline/realism); a clean quant CONFIRMED requires DSR and arrives at M3.

Calma occupies the VALIDITY layer — "is the claim real and robust" — above reproducibility platforms (same code → same number), beside integrity SaaS (prose, no execution), downstream of agent benchmarks (capability, no product). VERSION-GATED, the same way the SKILL.md description is: at M1 the identity is "independently RECOMPUTE-and-DIFF a result against its claim (canonical recipe + independently-bound input + baseline/comparative-validity check)"; the FULL multi-family validity layer (leakage re-run, overfitting/DSR, realism breadth) is explicitly deferred to M3. Any sentence using "validity layer", "five families", or "prove-it-or-break-it" MUST carry the installed-milestone gate — not only the frontmatter description (anti-overclaim invariant). Novelty is stated DEFENSIBLY: "we are not aware of a shipped tool that executes-and-diffs against a human claim across these failure families"; competitive-landscape verification is a tracked M2/M3 item before any first-mover phrasing. ## 4. Core input model (verify.yaml)

ONE declarative artifact contract, verify.yaml, AUTO-DRAFTED by draft_contract.py and shown back as a single BATCHED plain-language confirm screen (not raw YAML, not N sequential prompts). A concrete verify.schema.json plus a fully-populated BTC fixture verify.yaml ship in assets/ as the single source of truth all scripts validate against. Fields (language quarantined to run + env): (1) run — entrypoint (shell | Make | run.sh | papermill notebook | Dockerfile CMD); precedence run.sh > Makefile target > Dockerfile CMD > README code block. (2) env — detected spec + ecosystem adapter recorded as a 4-GATE capability record {restore | run | emit-raw | bind}; carries a per-package install-script-risk row and a dependency-trust classification per ecosystem (PyPI: OSV+typosquat+wheel-only-resolvable; CRAN/Bioconductor: known-registry+pin; conda: channel allowlist; Julia: General-registry vs git-url; vendored C++/no-manifest → untrusted-by-default). A Python package that MUST build from source is flagged source-build → tier-escalation (no per-package script-deny exists for PEP 517 build backends). (3) artifacts — output files each with a SEMANTICALLY-TYPED column schema (tag ∈ {return,price,score,prob,label,weight,timestamp,group} + dtype+unit+periodicity) + per-column na_values/na_policy ∈ {drop,error,zero-fill} + language-aware deserialization rules (R factor↔label round-trip, NA/NaN/Missing canonicalization, 1-based index-column drop, timestamp unit+tz inference) and CLAIMED values; binding independently sanity-checked (column type/range/name AND ROW-integrity: duplicate-row, monotonic-order for ordered tags, row-count-vs-periodicity×span) AND assigned a graded state {independently-bound | plausibly-bound | author-asserted}; ANY cross-language deserialization ambiguity or failed row-integrity check forces at most plausibly-bound. Only an independently-bound input may underpin REFUTED on ANY axis. HARD RULE: recompute only from csv/parquet/json/npy/arrow — never a rendered notebook cell; run-produced headline artifacts must be RE-EMITTED by the verified entrypoint inside the sandbox (stamped re-emitted-by-entrypoint vs author-supplied-claim-path-unverified; the latter caps at CONFIRMED-WITH-CAVEATS). (4) metrics — first-class metric-SPECIFICATION; canonical recipe wins by default but each recipe ships a CO-EQUAL accepted-convention SET (Sharpe rf in/out, geometric/arithmetic annualization, periodicity 252/365/52, macro/micro/weighted AUC/F1) with a per-metric set-maturity flag {reviewed|provisional} — a convention IN a reviewed set yields clean CONFIRMED with the convention NAMED; an out-of-set or provisional-set recipe can only CAVEAT (never REFUTE) unless the conclusion flips under EVERY plausible convention in a neighborhood AND the author has confirmed the convention is not theirs. (5) controls — a multi-knob DETERMINISM CONFIGURATION CONTRACT (see pipeline): per-library seeds + worker_init_fn, use_deterministic_algorithms, cudnn flags, CUBLAS_WORKSPACE_CONFIG, TF enable_op_determinism, BLAS/OMP/MKL thread pins set BEFORE interpreter import, batch-shape pin, network off, device/precision, container on/off, K policy. (6) split — optional split-injection hook; vocabulary cli-flag | env-var | config-key | split-config-file | papermill-parameters-cell | consented-diffed-split-patch (compiled/no-seam), with declarability AND injectability measured PER LANGUAGE × seam. (7) selection_axes — declared seeds tried, hyperparameter grid, windows/universes/cohorts, baselines[] + a prompted selection_scope {only-evaluated | best-of-N | best-of-undisclosed} for undeclared cross-unit selection. (8) domain — quant|ml-benchmark|ab-test|stats|generic. Inline CLI claim ("/calma ./repo \"Sharpe 2.1\"") is pinned for CONFIRMED/CAVEAT but a REFUTED on it requires the same target-confirmation as in-repo (reconcile to an artifact value OR explicit one-tap confirm of number+metric+units), else INCONCLUSIVE. Confirmed bindings + pinned conventions PERSIST into the committed verify.yaml (no new file); re-runs and CI show only DIFFS from the committed contract, so low-tap holds across runs/machines and an absent verify.yaml is the only full-interrogation case. The contract is the sole source of language-agnosticism.

## 5. Architecture

A Claude Code skill packaged like paper-audit (SKILL.md TOC body <500 lines + references/ + scripts/ + assets/), installed via marketplace or .claude/skills/calma/, invoked as /calma (auto-fire SUGGESTION-only behind a consent-token interlock — run_hermetic.py and draft_contract.py refuse install/run unless a recorded consent token is present). Two layers. (1) ORCHESTRATION cloned from paper-audit as CONTROL FLOW + package shape (audit→fix→re-audit loop, report/fix modes, status lifecycle, anti-patterns) — but the gate PREDICATE (strict lattice + findings-floor), _validate() RULES (verdict re-derivation, binding gate on all REFUTED axes, non-waivable→non-clean, reverify-kind dispatch), and a net-new claim-verdict lifecycle are EXTENDED tested code: claims carry verdict_status {provisional,stable,locked} + verdict_history; a verdict flip under STRICTLY-IMPROVED inputs (higher tier, controlled band, narrower budget) is EXPECTED monotone convergence, never a regression or an identical-input re-draw; 'locked' = strongest-available-tier + coverage-met. (2) A net-new EXECUTION substrate where essentially all engineering/feasibility risk lives — draft_contract.py, run_hermetic.py (verified isolated install+run+startup+shim+determinism+repair), recompute.py (reference-deterministic over pinned libm), compare.py (statistically-calibrated dual-class budget + claim-SE), consistency.py, stats_engine.py, attest.py, and the SHARED verdict.py (imported by compare.py AND ledger.py so the label and its re-derivation are one function) — each with a specified argv + JSON --out schema + exit-code table (references/script-interfaces.md), invoked by the model via Bash (model READS outputs, never computes). On-disk workspace: .calma/<run-id>/{verify.yaml, ledger.json, runs/baseline/*, runs/controlled/*, runs/k*/, band.json, recompute.json, diff.json, manifest.json, consent.token}, each intermediate file schema'd as the inter-script wire format. The substrate's defining responsibilities beyond reproduction are VERIFIED SECURITY ISOLATION of untrusted code AND dependencies (install + run + interpreter startup), a CANONICAL-RECIPE authority over a graded input binding, a reference-deterministic recompute, and an FP-safe statistically-calibrated tolerance with deterministic verdict-enum emission. High-freedom PROSE governs judgement; low-freedom deterministic SCRIPTS own every fragile step and the verdict label itself.

## 6. Pipeline

- **0. Discover + draft contract + graded input-binding + supply-chain triage**
  - *purpose:* Turn a bare repo pointer into a confirmed verify.yaml with zero config, classify trust+dependency posture per ecosystem, assign a GRADED binding state per metric input, and decompose UNVERIFIABLE by which of the 4 gates fails.
  - *how:* draft_contract.py (sub-steps each with defined I/O): 4-gate env-adapter record (restore|run|emit-raw|bind) maps lockfile→ecosystem→restore command AND reports which gate fails; entrypoint precedence; artifact discovery (committed → bind now; run-produced → record EXPECTED schema + require re-emit-by-entrypoint, verify post-run); language-aware deserialization + semantic-tag inference (dtype+range+name-regex→tag) with column AND row-integrity sanity checks assigning {independently-bound|plausibly-bound|author-asserted}; canonical-recipe match (author recipe captured as 2nd interpretation; out-of-set/provisional → CAVEAT-only); split-hook detect + injectability test; LLM-driven claim-quad extraction (model PROPOSES) feeding the deterministic span-class + headline-confidence classifier (below-threshold headline-ness defaults to NON-headline); per-ecosystem dependency-trust + supply-chain pre-scan (OSV/pip-audit + typosquat) BEFORE any install; wheel-only-resolvability + hash-closure check. Read-only; NEVER installs or runs without a recorded consent token (hard interlock — auto-fire only DRAFTS and PRINTS the suggested command). On first /calma invocation runs `calma doctor` silently (cached), surfacing degraded-capability + the single enabling command before any wait.
  - *outputs:* Confirmed verify.yaml + initialized ledger.json (claims[] with verdict objects + findings[] with claim_id FK + scope + selection_scope) + trust/dependency-trust posture + per-metric recipe-authority + set-maturity + graded input-binding + per-gate UNVERIFIABLE decomposition + a pre-install supply-chain finding set + a consent token (absent until user confirms).
- **1. Zero-cost deterministic gates**
  - *purpose:* Catch arithmetic-impossible claims and obvious split-hygiene defects before any execution; deliver a guaranteed-non-empty first value even when no runtime/artifact/env is available.
  - *how:* consistency.py (net-new): GRIM/SPRITE/statcheck fire ONLY when the binding's semantic tag implies a DISCRETE/integer-granular measure with known item count (survey/label-count tables) — NEVER on continuous tags (return/price/score/prob/weight), where only decimal-precision/rounding plausibility is valid (granularity assumption stamped); capped at CAVEAT/finding-for-author, never standalone REFUTED. Plus unit/magnitude/sign + binding sanity + recipe-divergence pre-checks; class-prevalence/base-rate stamp; an evaluation-prevalence-vs-natural-prevalence check (default-firing CAVEAT when the eval set is suspiciously balanced and no deployment base rate is declared); a Python/sklearn static split-hygiene scan stamped SECONDARY with its ~0.68 recall and Python-only scope. Always returns at least these findings + the claim-grounding echo ('here is the claim I'd check and the one input I need') as the cold-start first value.
  - *outputs:* Cheap findings; an early static-only INCONCLUSIVE verdict with a concrete unblock when no runtime is available; the guaranteed first-value payload.
- **2. Verified isolated install + isolated run + CONTROLLED determinism + sandboxed repair**
  - *purpose:* Install deps and run the entrypoint inside the strongest VERIFIED isolation tier, CONTROL nondeterminism before measuring it, and bound run-to-run spread so REFUTED is reachable without manufacturing false REFUTEDs.
  - *how:* run_hermetic.py treats INSTALL, every interpreter STARTUP, the determinism-shim/enumeration import, and RUN as untrusted-code phases under the SAME tier (Claude Code native sandbox/microVM > gVisor > rootless+seccomp container > vetted+VERIFIED macOS Seatbelt > REFUSE). Tier-0 is the Claude Code native sandbox (Seatbelt/bubblewrap + proxy allowlist) — always available, so REFUTED-reachability never hinges on Docker liveness. Install hardening: wheel-only (--only-binary=:all:) HARD default — any source-build is a tier-escalation event (container/VM; on a no-daemon host → INCONCLUSIVE), because PEP 517 build backends have NO deny-and-proceed hook (wheel-only is the ONLY Python install-script denial); npm/pnpm-style per-package install-script deny+consent applies only where a hook exists; R/CRAN deserialization-on-load, conda link-scripts, Julia build.jl, Cargo build.rs have NO per-package gate → FULL sandbox for the whole install. Offline-first install: Calma builds a hash-pinned wheelhouse in an isolated resolve phase (so author hash-absence doesn't force the strong tier when Calma can self-lock), installs --no-index --find-links=<wheelhouse> --require-hashes --only-binary=:all: with network fully OFF; 'network needed during install' is a tier-escalation trigger to container+proxy, never a silent network-on Seatbelt install. Author-supplied hashes are stamped 'pinned to author hashes (not independently verified)' distinct from registry/OSV-clean; a recompute depending on an author-pinned dep that DIVERGES from the canonical-registry version down-ranks a REFUTED to INCONCLUSIVE. Network: default-deny egress at the boundary (container netns/--network none; Seatbelt deny network*); the rare must-fetch case uses a CONNECT-only proxy with SNI+resolved-IP pinning to the official index (no TLS termination/MITM CA → no credential exposure), rejects IP-literal/DoH egress, and IGNORES repo-supplied --index-url/pip.conf (the dependency-confusion pivot); run-phase network OFF unconditionally; install-network + proxy-used posture stamped. Filesystem policy is DENY-BY-DEFAULT READ with an explicit minimal allowlist (repo dir, wheelhouse, scratch, interpreter prefix) so unknown secret paths are denied by construction; the shipped .sb Seatbelt profile (assets/) is verified by a POSITIVE-CONTROL self-test: a planted .pth/sitecustomize attempting denied egress+secret-read must be BLOCKED, and the guest mount table is read fail-closed if $HOME or any host path beyond scratch is visible, else 'host-tier = NOT isolated'. macOS DoS enforcement is concrete: run the untrusted tree in its own session/process-group, kill the WHOLE group + sweep re-parented strays on timeout, cap spawned-process count by monitoring (not RLIMIT_NPROC), enforce disk via a fixed-size scratch volume (not ulimit -f); any kill/timeout/OOM-sweep → INCONCLUSIVE. A first-class consented Calma-managed VM path (lima/krunkit/Apple Virtualization microVM) survives sandbox-exec deprecation; `calma doctor` exercises an ACTUAL deny (egress+secret-read) at runtime and fails closed. DETERMINISM CONFIG CONTRACT applied INSIDE the verified boundary BEFORE measuring: framework knobs injected via a Calma-authored sitecustomize.py on PYTHONPATH (NOT a site-packages .pth) + thread/BLAS env set before exec; the shim is a determinism AID, never a containment control (stamped as such). Mode is stamped: CONTROLLED-TO-BIT only when use_deterministic_algorithms(True) succeeds with ZERO warn_only ops AND the enumerated nondeterministic-op set is empty (a STRUCTURAL determinism proof; K=2-3 only SANITY-CHECKS it); MEASURED-BAND when any op survives; UNCONTROLLED for non-Python (env-vars only) — supports ONLY INCONCLUSIVE. Atomic scatter/segment/index reductions are unbounded-magnitude by default → INCONCLUSIVE unless a forward bound is computed. Repair loop: EXECUTE→CLASSIFY→bounded REPAIR(3)→RE-EXECUTE under one global budget; the as-shipped band is frozen ONLY for the laundering/value-drift guard (a result-affecting dependency/code repair drifting the center outside it spawns a reproducibility CAVEAT). Applying the determinism config is NOT a laundering repair; the determinism-config-induced shift is stamped separately ('forcing determinism moved Sharpe 2.10→2.08') and the laundering guard compares like-for-like only.
  - *outputs:* Per-phase isolated raw artifacts (incl. frozen as-shipped band, controlled band, re-emitted headline artifact) + stamped isolation/install-network/proxy/hermeticity tiers + hardware/env fingerprint (CPU/GPU model, CUDA/cuDNN/driver, BLAS vendor+version, framework version, OS, thread counts) + enumerated nondeterministic ops with bounded/unbounded flags + determinism_mode + per-phase exit codes + resolved SBOM (name+version+hash+trust-class) + classified repair log + consent-token check.
- **3. Recompute-from-raw + statistically-calibrated tolerance diff**
  - *purpose:* Does the shipped code produce the claimed number, by Calma's canonical definition over an independently-bound input, distinguishably from the claim's own sampling noise?
  - *how:* recompute.py reads RAW (re-emitted) outputs and recomputes via the canonical recipe over the typed binding in Calma's REFERENCE-DETERMINISTIC path: float64 always; Welford/two-pass variance; math.fsum/Kahan/Neumaier summation; log-domain cumulative products via a PINNED correctly-rounded libm (CORE-MATH/crlibm or mpmath at fixed precision — libm log/exp/pow/sqrt are NOT correctly-rounded and differ across platforms); FP-contraction/FMA OFF in reference kernels; NO bare np.sum/np.mean/np.std/np.var/np.cumprod in any recipe (lint+AST test enforced — SIMD-width-dependent). Guarantee is downgraded honestly: 'bit-identical on the same Python/NumPy build+libm (hard gate); reproducible within a STAMPED reference-ULP envelope across builds/platforms', and that non-zero reference-ULP envelope feeds the recompute-spread term (never assumed zero). NaN/Inf/sentinel/non-finite or degenerate recompute → INCONCLUSIVE 'data-cleaning policy undetermined', NEVER REFUTED. Calma's recompute runs K times on the same raw artifact to capture residual numeric spread. compare.py separates budgets: REPRODUCIBILITY band (author-entrypoint run-to-run spread; gates 'did it reproduce' + laundering); and the RECOMPUTE-AGREEMENT budget = a DETERMINISTIC worst-case component (per-metric forward-error floor + summation-order floor + reference-ULP envelope, combined by MAX/addition) PLUS a RANDOM component (Calma-recompute-spread ⊕ cross-stack random spread, quadrature only among independent random pieces) — the two CLASSES added, never one quadrature over a worst-case bound and an SD — PLUS, for any estimator with one, the claimed statistic's SAMPLING standard error (Sharpe SE=sqrt((1+0.5·SR²)/T) skew/kurtosis-corrected; DeLong for AUC; cluster/robust for A/B), reused from stats_engine. A per-metric-class MINIMUM FLOOR (calibrated offline) bounds the budget below so an anomalously small self-measured cross-stack spread cannot tighten it into a false REFUTED. Cross-stack budget = floor(forward-error bound) combined with an UPPER confidence bound on {BLAS×thread×summation, same-language oracle} dispersion (not a bare max over noisy samples); the same-language oracle, when env won't restore, may be a tiny pinned single-metric container, else the budget is wider and more likely INCONCLUSIVE. Cross-stack ATTRIBUTION is a POSITIVE test, not a size comparison: a gap is cross-stack only if it SHRINKS under higher precision (float64→mpmath) and/or matches the same-language oracle's own divergence; a precision-INVARIANT gap is a candidate definition error (not down-ranked); un-attributable → INCONCLUSIVE. The REFUTED-on-magnitude rule requires the gap to EXCEED the effective budget AND the claim+recompute to be STATISTICALLY DISTINGUISHABLE (claim outside the recompute's CI). Path-dependence is a first-class CONDITION (multimodal band, argmax/sort/threshold metrics like max-drawdown, near-tie between top-two candidates) → INCONCLUSIVE/CAVEAT, never magnitude-REFUTE. The verdict ENUM is emitted by the deterministic verdict() pure function (shared verdict.py, imported by compare.py AND ledger.py) from the FULL vector {gap, budget_components, effective_budget, band_confidence, cross_stack, flip_distance, binding_status, isolation_tier, exit_codes, determinism_mode, nondeterministic_ops, path_dependent} — never the model.
  - *outputs:* recomputed-vs-claimed diff table (canonical + author rows, named convention) + budget breakdown + reproduction_status and inferential_status as separate stamped sub-results + verdict_inputs (= verdict()'s full argument vector) persisted for the re-derivation check.

## 8. Verdict model

NEVER a bare pass/fail. Per-CLAIM, with reproduction_status (did it reproduce within the determinism band) and inferential_status (does it survive DSR/realism/baseline) reported as SEPARATE stamped sub-results; the prose names which dimension drove a non-CONFIRMED outcome ('reproduced exactly, fails deflated-Sharpe at N=240' vs 'did not reproduce within the band'). AXIS 1 enum, emitted ONLY by the shared deterministic verdict() pure function over its FULL input vector {gap, budget_components, effective_budget, band_confidence, cross_stack, flip_distance, binding_status, isolation_tier, exit_codes, determinism_mode, nondeterministic_ops, path_dependent}: CONFIRMED (recomputed within the recompute-agreement budget by the canonical recipe over an independently-bound input, sign-agrees, survives the family review) / CONFIRMED-WITH-CAVEATS (holds but narrows — cited cost assumptions, thin slice, declared/floored N, accepted-set author convention, plausibly-bound input, cross-stack-attributable difference, hardware-sensitive headline, no baseline declared, author-supplied-not-re-emitted artifact, best-of-undisclosed selection, provisional convention set) / REFUTED (gap EXCEEDS the recompute-agreement budget by the calibrated margin AND the claim lies OUTSIDE the recomputed estimate's confidence interval — statistically distinguishable, not merely numerically far — AND, for a conclusion-flip, the threshold lies outside the parameter's ONE-SIDED sampling CI oriented to the flip; OR a baseline-edge whose zero/sign-flip lies outside the recomputed edge's CI; OR an out-of-set/definitionally-invalid recipe that flips the conclusion under EVERY neighborhood convention with the author having confirmed it is not theirs — recipe non-canonicality alone never REFUTES, an out-of-set-but-plausible convention only CAVEATs; REQUIRES every dependent input independently-bound AND a structured reproduction whose `expected` assertion is validated against the manifest) / INCONCLUSIVE-or-NOT-VERIFIABLE (artifact not machine-readable, NaN/Inf/degenerate recompute, band low-coverage/heavy-tailed/small-K/uncontrolled-and-insufficient-K and no container, unbounded-magnitude nondeterministic op present, env un-restorable, no isolation primitive, input not independently-bound, pipeline-leakage un-rerunnable, path-dependent near-tie, resource-kill/timeout/OOM, masking-too-wide or un-attributable cross-stack budget, divergent author-pinned dependency, claim target unconfirmed — explicitly NOT a refutation). AXIS 2: CVSS severity. AXIS 3: deterministic vs judgement. REPO-LEVEL: FP-aware worst-of-claims — only a deterministic, confidence-OK REFUTED on a CONSERVATIVELY-classified HEADLINE claim (below-threshold headline-confidence defaults to NON-headline) sets repo REFUTED; a non-waivable REFUTED on ANY claim forces a non-clean repo_verdict (MIXED/CONTESTED) and a non-waivable blocker finding so the headline rule can only downgrade the LABEL, never upgrade into a gate-passing state; non-headline/judgement/low-confidence REFUTED rolls to CONFIRMED-WITH-CAVEATS + flagged claim; with no deterministic headline REFUTED the headline reads CONTESTED/REVIEW; MIXED is a declared non-CONFIRMED repo value with a per-claim breakdown. Family-wise false-REFUTED control is a CALIBRATED per-claim FP rate (from the FP-guard corpus) aggregated by a UNION BOUND over HEADLINE claims only (each headline margin set so empirical FP ≤ alpha/H), corrected over each claim's INTRINSIC families/conventions (not the repo-wide claim count) with a per-claim absolute-margin floor — explicitly NOT a p-value/Bonferroni correction; H + aggregation method stamped; missed-REFUTED reported co-equally. CRITICAL INVARIANTS: availability/isolation/binding/band-coverage/resource-kill/determinism_mode are SEPARATE axes from 'does the claim survive' and can never produce REFUTED; DSR/PSR is NECESSARY-NOT-SUFFICIENT (clears the OVERFITTING axis only, never grants a magnitude CONFIRM); the conclusion-FLIP and baseline-edge paths obey the parameter-CI guard (machine-noise band never underwrites a conclusion-flip); env-repair value-drift over the frozen as-shipped band is a reproducibility finding (the verdict reads the controlled band); objective reproduction-backed REFUTED is non-waivable to clean CONFIRMED. A quant CONFIRMED additionally requires a declared/floored N + DSR (with spectral-K_eff/Var[SR]/T) + one-sided Harvey-Liu f(N,method) + realism deflators + baseline edge — and is UNREACHABLE until M3 (at M1 quant reaches REFUTED/CAVEAT/INCONCLUSIVE only). Every verdict stamps SCOPE + 'what would change this verdict.'

## 9. Language-agnostic strategy

Language is confined to run + env and never leaks into verification logic: Calma re-runs the user's entrypoint as a black box, reads emitted machine-readable files (re-emitted by the verified entrypoint, not author-supplied committed files for headline claims), and recomputes in its OWN reference-deterministic Python layer over a graded typed binding with LANGUAGE-AWARE deserialization (R factor↔label, NA/NaN/Missing canonicalization, 1-based index drop, timestamp unit+tz inference; any ambiguity forces plausibly-bound). Env restore is an ecosystem-dispatched 4-GATE capability record (restore|run|emit-raw|bind) over uv/requirements, renv/DESCRIPTION, Manifest.toml, CMake/Make/Dockerfile, pixi — but env RESTORE is itself untrusted-code execution behind the SAME verified isolation tier as the run, and for ecosystems with no per-package install-script gate (R/CRAN deserialization-on-load CVE-2024-27322, conda link-scripts, Julia build.jl, Cargo build.rs) AND for Python SOURCE builds (PEP 517 backends have no deny-and-proceed) the WHOLE install is forced to full sandbox; no host-tier source-build install path exists. Determinism control is a stamped scope dimension (full Python/torch | partial thread-pinning | none); portable knobs (OMP/BLAS thread pins=1 set before import, platform RNG seed env, single-threaded reductions) apply cross-language, the wide UNCONTROLLED band is reserved for the genuine residual and supports only INCONCLUSIVE — BUT REFUTED-reachability is DECOUPLED from the determinism tier: a non-Python uncontrolled band can still REFUTE when the gap exceeds the coverage-inflated, heavy-tail-screened band by the calibrated fraud-multiple M and path-dependence is absent (the tier governs WIDTH, not eligibility), with per-language REFUTED-reachability published. The cross-stack numeric budget is EMPIRICAL per artifact (floor on a forward-error bound + an upper CI on {BLAS×thread×summation, same-language oracle} dispersion, attributed positively by precision-shrink/oracle-match), not a per-language constant; when the same-language oracle is unreachable the budget is wider and the metric is likelier INCONCLUSIVE (published per-language). The leakage re-run reaches across languages via the DECLARED hook or, where no non-editing seam exists, a consented diffed sandbox-applied manifest-stamped split-patch (carved out from the no-source-edit invariant), with declarability/injectability measured per language×seam in three buckets; static fallback is Python-only and non-Python no-seam repos are stamped 'leakage NOT RE-RUN.' Notebooks are a named execution class split BY KERNEL (ipykernel|IRkernel|IJulia): the persist-to-artifact and parameters-cell affordances are Python-kernel-first; R/Julia notebooks fall back to the same no-non-editing-seam and emit-raw-required rules as their non-notebook counterparts. The active clock probe is dynamic-runtime-only. The honest output is a published per-language 4-gate served-fraction + REFUTED-reachability matrix, not a claim of uniform adapter parity — C++/GPU/build-from-source frequently land UNVERIFIABLE and the positioning says so.

## 10. Flagship quant path

Quant backtest verification is a specialization of the one general engine; domain=quant (auto-detected from trades.csv/backtest cues, suggestion-only) selects references/backtest.md, the quant realism template, and the quant baseline set. AT M1 the flagship delivers the BREAK-IT half only: a backtest can reach REFUTED (baseline/realism), CONFIRMED-WITH-CAVEATS, or INCONCLUSIVE — a clean CONFIRMED is structurally UNREACHABLE until M3 because the DSR precondition cannot be computed before stats_engine ships; the tagline's 'prove it' half for quant is an M3 capability and the copy says so. A backtest CONFIRMED (M3+) is gated and cannot print until: (1) recompute-from-raw of Sharpe + max-drawdown from the RE-EMITTED machine-readable trade log via Calma's reference-deterministic canonical recipe over an independently-bound returns/PnL column (vol-plausible, centered, periodicity matched via a DOCUMENTED periodicity binding — 252 equities / 365 crypto / 52 weekly, NOT a silent constant; a mis-tagged or row-corrupted column is a data-integrity finding capping at plausibly-bound) matches the claim within the recompute-agreement budget that INCLUDES the Sharpe sampling SE=sqrt((1+0.5·SR²)/T) AND the gap is statistically distinguishable AND judged against the controlled, coverage-stated band (a fraud-grade inflation REFUTES even on an L0/L1 GPU/BLAS run when the gap exceeds the heavy-tail-screened band by the calibrated fraud-multiple M — defined as gap ≥ M·combined_sigma where combined_sigma includes the sampling SE, NOT a 0.25/beat-BH heuristic; a gap inside the band, within one SE, or a low-coverage/uncontrolled-insufficient-K band with no container reads INCONCLUSIVE); (2) a declared or evidence-floored trial count (spectral/participation-ratio K_eff, never the literal config count, never auto-inferred, DSR computed across a plausible N range when N is author-asserted) with Var[SR] across trials and T present clears DSR (PSR-core, skew/kurtosis-corrected, NECESSARY-NOT-SUFFICIENT) + one-sided Harvey-Liu required-t = f(N,method) (PBO@S=16 only with the full per-period score matrix + band, blocked under return autocorrelation, Bayesian cross-check on instability, else DSR; short-T low-power DSR that fails to reject → INCONCLUSIVE); (3) realism deflators applied as a sensitivity sweep with a Sharpe-SE confidence ribbon, the net-of-realism Sharpe surviving across a defensible region under cited defaults via the one-sided parameter-CI flip guard; AND (4) a BASELINE edge whose zero/sign-flip lies OUTSIDE the recomputed edge's CI (a CI straddling zero → CAVEAT, never REFUTED). The shareable artifact is the numeric collapse with a reproduction ('recomputed OOS Sharpe 0.4 net-of-costs vs claimed in-sample 2.1; underperforms buy-and-hold OOS on the baseline axis; overfitting/DSR verdict arrives at M3'). ## 11. Ease-of-use design

- Strictly-progressive verdict render for non-experts: line 1 = a plain-words trust signal + traffic-light/0-100 confidence; line 2 = the one thing most limiting it; the full scope block, tiers, bands, binding states all collapsed behind 'show full breakdown' — default surface capped at ~3 lines regardless of how many families/tiers stamped. The exact copy for all four verdict classes and the repo states (CONTESTED/MIXED/OVERRIDDEN, renamed if non-experts misread them) lives in references/verdict-rubric.md and is gated on a comprehension test (can a non-quant answer 'should I trust this?' from the collapsed view).
- Zero-config common case + a single batched confirm screen presenting inferred bindings as plain-language assertions ('I think column "ret" is your daily strategy returns — yes / no / something else'), never raw YAML; unknowable fields (rf input, periodicity, na-policy) default to the conservative author-asserted stamp without asking; confirmed bindings + pinned conventions PERSIST into the committed verify.yaml so re-runs/CI/teammates see only DIFFS and skip re-confirmation (absent verify.yaml is the only full-interrogation case).
- Legitimate convention pluralism does not cry wolf: a convention in a REVIEWED co-equal set yields clean CONFIRMED with the convention NAMED and a one-tap 'this is my intended convention, remember it' that pins it for the repo; an out-of-set or provisional-set convention can only CAVEAT (never REFUTE) — the 252-vs-365 case is treated as a confirm-the-intended-convention prompt, not an automatic ERROR.
- Every INCONCLUSIVE/refusal is a guided next-step screen routed BY WHO CAN ACT: user-self-serve actions (start daemon, provide lockfile, install faketime) get a copy-paste command or an offer to do it; author-only actions (emit a machine-readable artifact, declare a split hook) are reframed as a one-line forwardable message to the author, never a CLI instruction to the current reader; the INCONCLUSIVE-unblock screen is itself in the comprehension-test gate.
- Inline CLI claim is the fastest path (pinned for CONFIRMED/CAVEAT) but a REFUTED on it requires the same target-confirmation as in-repo: even on the trusted inline path, when the recompute lands in REFUTED territory, one confirmation of the claimed number+metric+units fires ('you stated Sharpe 2.1; I recompute 0.4 — confirm the claim is Sharpe 2.1 daily-annualized') before REFUTED, else INCONCLUSIVE.
- Read-only by default with --dry-run; the env-repair loop is sandboxed and any repair that moves the metric outside the frozen as-shipped band is surfaced as a reproducibility finding, never silently applied; Docker is never required for your OWN trusted code WHEN the Seatbelt profile passes its self-test (else an isolation CAVEAT, never a silent host-tier stamp), with the untrusted-third-party container requirement and the dependency-trust install rule stated adjacently so the constraints are never dropped.
- Streamed progress ('classified env-drift → built offline wheelhouse → restored uv.lock → measured controlled band (stable, K=3 sanity) → re-running under purged split') so the approval-gated full re-run is never a silent hang, with the 'stop and keep what we have' affordance during any consented long run.

## 18. Risks & mitigations

- **Install-time code execution (PEP 517/setup.py, conda link-scripts, Cargo build.rs, R deserialization-on-load CVE-2024-27322, .pth at interpreter startup) owns the host before recompute runs; the build runbook descopes install isolation from M1 ('native host sandbox only') while the design lists it in-MVP — the first thing the build session does is pip-install untrusted code with no install-time isolation.**
  - *mitigation:* Reconcile the two docs BEFORE any build: install AND every interpreter startup AND the determinism-shim import are untrusted code under the SAME verified tier as the run; M1's definition-of-done INCLUDES the minimal install-isolation primitives (Seatbelt deny-by-default-READ profile + network-off-during-install + wheel-only default + offline hash-pinned wheelhouse + deny-install-scripts-where-a-hook-exists). M1 may NOT install/run any untrusted third-party repo on the bare host: the flagship demo runs a USER-AUTHORED Seatbelt-tier fixture, and run_hermetic refuses to install third-party deps unless the verified tier is established. Update the runbook to match.
- **Seatbelt does not contain install-time native code execution: it confines writes/network on the HOST kernel at the user UID; to install, it must allow write+exec to site-packages, so a malicious sdist build backend or wheel .pth executes with full user privileges. 'Trusted repo + one untrusted transitive dep' = host RCE in a write-confined box; the trust split keyed on REPO authorship is unsound for install.**
  - *mitigation:* Gate install-time risk on DEPENDENCY trust, not repo authorship: any install pulling non-vendored/non-hash-pinned deps requires the container/VM tier OR an offline vendored wheelhouse of pre-vetted artifacts; pure-Seatbelt installs are allowed ONLY for a fully hash-pinned, wheel-only, vendored closure with install-scripts denied, and are stamped 'host-kernel shared, NOT escape-isolated — relies on egress+credential denial only' so the guarantee is never overstated.
- **Offline-install is unspecified and an egress proxy is unbuildable under Seatbelt (OS sandboxes give all-or-nothing network, no domain allowlist); on the MVP host (macOS, docker down) the only sound offline mechanism — a vendored wheelhouse — is nowhere specified, so a real install opens the network (exfiltration channel + determinism contaminant).**
  - *mitigation:* Specify a vendored offline-install substrate as the MVP default: Calma builds a hash-pinned wheelhouse in an isolated resolve phase, installs --no-index --find-links=<wheelhouse> --require-hashes --only-binary=:all: with network OFF at the Seatbelt boundary; 'network needed during install' escalates to container+proxy (down on MVP host → INCONCLUSIVE), never a silent network-on Seatbelt install. Document per-ecosystem offline caches (renv cache, conda pkgs, Julia depot, cargo vendor) and stamp install-network posture.
- **Hash-pinning gated on lockfile existence routes the COMMON case (no fully-hashed lockfile) to the strong tier, which is down on the MVP host → INCONCLUSIVE far more often than the 'fraud-grade inflations dwarf the band' optimism implies.**
  - *mitigation:* Calma self-locks: resolve + pin + hash the closure in an isolated resolve phase, then install offline from that, so repo hash-absence doesn't force the strong tier; add 'fraction of corpus repos with a wheel-only-installable hashable closure' to the M2 validation plan as a gating, published served-fraction metric; be explicit that on macOS-no-daemon, genuinely unhashable/source-build installs → INCONCLUSIVE.
- **PEP 517 build backends have no deny-and-proceed hook — 'deny-install-scripts' (npm-model) silently does nothing for the dominant Python source-build RCE; building from source IS executing the code.**
  - *mitigation:* State explicitly: for Python the ONLY install-script denial is wheel-only resolution; --only-binary=:all: is the HARD default; any source-build is a tier-escalation event (container/VM) and on the no-daemon host → INCONCLUSIVE, never a Seatbelt source build. Apply the same 'no per-package gate exists' rule already stated for R/conda/Julia/Cargo to Python SOURCE builds.
- **Asserted 'verified' controls are spoofable as worded: a credential-path denylist-probe can't prove ALL secret locations are blocked (allowlist-write+denylist-read is the wrong polarity); '.pth coverage' and 'no VM mounts $HOME' are asserted, not enforced.**
  - *mitigation:* Invert filesystem policy to DENY-BY-DEFAULT READ with a minimal allowlist (repo, wheelhouse, scratch, interpreter prefix) so unknown secret paths are denied by construction; run a POSITIVE control — a planted .pth/sitecustomize attempting denied egress+secret-read must be BLOCKED before stamping coverage; read the guest's ACTUAL mount table and fail-closed if $HOME or any host path beyond scratch is visible.
- **sandbox-exec is Apple-deprecated and is the ONLY isolation primitive on the MVP host; its removal vanishes the entire host boundary → 100% INCONCLUSIVE on the primary dev platform.**
  - *mitigation:* Ship a first-class consented Calma-managed VM path (lima/krunkit/Apple Virtualization microVM cold-start) as an MVP tier, not only colima-as-Docker-backend, so the boundary survives sandbox-exec removal; `calma doctor` exercises an ACTUAL deny (egress+secret-read) at runtime and fails closed rather than trusting binary presence.
- **macOS resource-exhaustion enforcement rests on absent/insufficient primitives: cgroups don't exist on macOS, per-process ulimit is defeated by a fork bomb, a parent wall-clock kill doesn't reap re-parented processes, disk quotas aren't ulimit-enforceable.**
  - *mitigation:* Run the untrusted tree in its own session/process-group and kill the WHOLE group + sweep re-parented strays on timeout; cap spawned-process count by monitoring (not RLIMIT_NPROC); enforce disk via a fixed-size scratch volume (APFS/disk-image quota); any kill/timeout/OOM-sweep → INCONCLUSIVE; where unguaranteeable on the bare host, require the VM tier for genuinely untrusted code and stamp DoS as best-effort.
- **The adaptive band was a min-max range with no coverage semantics on heavy-tailed/multimodal GPU nondeterminism; K=2-3 'agree' was being read as a tight band; the band's iid assumption is violated by same-session correlation.**
  - *mitigation:* Replace min-max with a distribution-free order-statistic tolerance bound at a stated (coverage, confidence); CONTROLLED-TO-BIT (structural determinism proof, K=2-3 sanity only) vs MEASURED-BAND (requires the coverage-driven K, e.g. ~59 for one-sided 95/95; if unaffordable → INCONCLUSIVE; small-K+uncontrolled ALWAYS → INCONCLUSIVE since heavy tails are undetectable at small K); force some COLD launches and stamp the independent-launch count; a cross-run drift/runs-test failure → INCONCLUSIVE; the band is same-session-conditional and never underwrites a cross-machine CONFIRMED; atomic scatter/segment/index reductions are unbounded-magnitude → INCONCLUSIVE unless a forward bound is computed.
- **The tolerance model captured numeric reproducibility but NOT the claimed statistic's own SAMPLING error, so compare.py could REFUTE on a gap within one standard error of a noisy small-T estimate (a numeric-agreement budget is not a statistical-agreement budget).**
  - *mitigation:* Add the claimed statistic's sampling SE as a MANDATORY budget component for any estimator with one (Sharpe skew/kurtosis-corrected, DeLong AUC, cluster/robust A/B), reusing stats_engine; REFUTED-on-magnitude requires the gap to exceed quadrature(numeric, cross-stack, author-precision) AND the claim to lie OUTSIDE the recompute's CI; conclusion-flips use a ONE-SIDED parameter CI, never the machine-noise band; the same guard applies to the baseline-edge difference-of-two-noisy-recomputes.
- **Reference recompute claimed bitwise cross-machine reproducibility, which is FALSE for any metric touching sqrt/log/exp (libm not correctly-rounded, FMA contraction, SIMD-width-dependent np reductions) — the M1.3 acceptance test would fail and the 'recompute spread ~0' premise is overstated.**
  - *mitigation:* Downgrade to 'bit-identical on the same build+libm (hard gate), reproducible within a STAMPED reference-ULP envelope across builds/platforms'; pin correctly-rounded transcendentals (CORE-MATH/crlibm or mpmath), compile reference kernels FMA/contraction-off, forbid bare np.sum/mean/std/var/cumprod in recipes (AST lint + SIMD-width test); feed the non-zero reference-ULP envelope as the recompute-spread floor so it is never assumed zero.
- **DSR/PSR was used as a CONFIRM gate (significance-vs-confidence inversion), specified on N alone, with a hardcoded two-sided ~2.0/3.0 Harvey-Liu threshold and a linkage-cluster K_eff that mis-estimates effective independence and violates DSR's iid-Gaussian-trials assumption.**
  - *mitigation:* DSR/PSR is NECESSARY-NOT-SUFFICIENT (clears the overfitting axis only; magnitude CONFIRM comes from Stage-3); it prints only with {spectral/participation-ratio K_eff, Var[SR], T, skew, kurtosis} present/floored, degrading to CAVEAT under strong-correlation/duplicate/anti-correlated pathology; Harvey-Liu required-t is ONE-SIDED f(N,method) from the skew/kurtosis-corrected PSR distribution (~1.645 at N=1); author-asserted N caps at CONFIRMED-WITH-CAVEATS and DSR is computed across an N range; unit-tested against mlfinlab/pypbo + duplicate/orthogonal K_eff vectors.
- **Repo-level FP control via 'Bonferroni-on-margins' is statistically ill-defined (no shared null, no p-values, dependent tests) and is gameable: padding the repo with trivial true claims tightens every margin and can suppress a real headline REFUTED.**
  - *mitigation:* Drop 'Bonferroni'; control false-REFUTED as a CALIBRATED per-claim FP rate from the FP-guard corpus, aggregated by a UNION BOUND over HEADLINE claims only (margin so empirical FP ≤ alpha/H), corrected over each claim's INTRINSIC families/conventions (not the repo-wide claim count) with a per-claim absolute-margin floor so padding can't aggregate away a headline REFUTED; report missed-REFUTED co-equally; stamp H + method.
